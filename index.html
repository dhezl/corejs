Objects

An object in JavaScript is basically a container. You can pretty much slug anything you want into an object:

  • values
  • arrays
  • functions
  • other objects

Objects

Objects are defined with curly braces, and properties are given as name:value pairs:

              
var user = {firstName: "Bob", lastName: "Moog", salutation: "Dr."};

Objects

Object properties are accessed using either dot syntax (preferred in most cases), or array notation:

              
var user = {firstName: "Bob", lastName: "Moog", salutation: "Dr."};
console.log(user.lastName); // returns "Moog"
console.log(user[lastName]); // returns "Moog"

Objects

When do I use dot or brackets?

Generally speaking, dot is cleaner, easier to read, and is preferred. Array-style access is used when you need special characters or arithmetic:

              
var startingPoint = 0;
var user = {firstName: "Bob", lastName: "Moog", salutation: "Dr."};
console.log(user[startingPoint + 1]); // returns "Moog"

Objects

Functions may be accessed inside of objects in two ways, as well:

Accessing functions with the invocation operator will return the result of that function. Doing so without the invocation operator will return the function definition.

              
var startingPoint = 0;
var user = {firstName: "Bob", sayHello: function(){ console.log("Hi there!"); } };
console.log(user.sayHello()); // returns "Hi, there!"
console.log(user.sayHello); // returns the function itself

TRUTHINESS

A really cool aspect of JavaScript is the fact that data types can be implicitly coerced from other data types, depending on usage.

This is most commonly used when coercing a value to a boolean. This is known as the value's truthiness.

TRUTHINESS

If non-boolean values are evaluated as booleans in JavaScript, a check for truthiness is being performed. Any non-zero value will return true. In this case, the value is said to be truthy. This can be very useful when checking for the presence of a property, but look out—any non-zero returns as truthy; this means that numbers have their truthiness evaluated based on their absolute value!

If the coerced boolean comes back as false, then the value is said to be falsy.

TRUTHINESS

              
var myArray1 = ["foo", "bar", "baz"];
var myArray2 = [];

console.log(myArray1.length); // outputs 3
if(myArray1.length){console.log("true")}; // outputs true
if(myArray2.length){console.log("true")}; // no output
if(myArray1.indexOf["tyrannosaurus"]){console.log("true")}; // outputs true

Conditional Logic

Simple conditionals form the basis of logic in JavaScript. This is written fairly intuitively, with two or more conditions to be met, each with a set of curly braces containing the appropriate actions to take:

              
var thinkingOfANumber = 10;
if (thinkingOfANumber <= 10) {
  console.log("I am thinking of a number from 1-10!");
} else {
  console.log("I am thinking of a number bigger than 10!");
}
// returns I am thinking of a number from 1-10!

Conditional Logic

You can chain as many if conditions as you want, really:

              
var thinkingOfANumber = 10;
if (thinkingOfANumber === 1) {
  console.log("My number is 1!");
} else if (thinkingOfANumber <= 10) {
  console.log("I am thinking of a number from 1-10!");
} else {
  console.log("I am thinking of a number bigger than 10!");
}
// returns I am thinking of a number from 1-10!

Conditional Logic

Beware doing that too much, though, or nesting it too deep. If your conditional logic gets too complex, that's a red flag.

Shore it up by breaking it into discrete chunks.

Conditional Logic

Switch statements are another way to write long conditionals:

              
var thinkingOfANumber = 10;
switch(thinkingOfANumber) {
  case 1:
    console.log("your number is 1");
    break;
  case 2:
    console.log("your number is 2");
    break
  default:
    console.log("i don't care what your number is");
}
// returns i don't care what your numnber is

Conditional Logic

Simple if/else statements can be written with what's called a ternary

              
if (2 + 2 === 4) {
  console.log("Congrats on stating the obvious.")  
} else {
  console.log("Did you go to public school in Texas?")
}
// returns Did you go to public school in Texas?

(2 + 2 === 4) ? console.log("correct") : console.log("incorrect");
// returns correct

Conditional Logic

...as an aside, ternaries can also be used for variable assignment:

              
var foo = (2 + 2 === 4) ? "correct" : "incorrect";
console.log(foo); // returns correct

for loops

The most common loop is the for loop:

              
for (var i = 1; i <= 10; i ++) {
  console.log(i);
}
// returns 12345678910

for loops

Note that the middle portion of such a loop is a condition which must be met for the loop to stop running. Typically, you see this as an expression, but—diving back into truthiness—any truthy value will work here:

              
var starks = ["Robb", "Sansa", "Arya", "Bran", "Rickon"];
for (var i = 0; starks[i]; i ++) {
  console.log(starks[i]);
}
// returns Robb Sansa Arya Bran Rickon

while loops

while loops simplify things, if you are are able to:

              
while (i <= 10) {
  console.log(i);
  i ++;
}
// returns 12345678910

while loops

In fact, you can think if a while loop as simply a for loop with the first and third statements trimmed:

              
var starks = ["Robb", "Sansa", "Arya", "Bran", "Rickon"];
var i = 0;

for (i; starks[i]; i ++) {
  console.log(starks[i]);
}

for (; starks[i]; ) {
  console.log(starks[i]);
  i ++;
}

while(starks[i]) {
  console.log(starks[i]);
  i ++;
}

Functions

Functions are a special type of object that is designed to perform a task. A function is executed when it is invoked by calling it using the () operator. 

              
var x = 5;

function logVariableValue(value) {
  console.log(value);
}

logVariableValue(x);

Functions can have parameters passed into them for consideration during execution. In defining a function, these variables are called parameters. The values themselves are known as arguments. 

              
var x = 5;

function logVariableValue(value) {
  console.log(value);
}

logVariableValue(x);

Functions return a value to their caller; this can be manual or automatic. If there is no return value specified, then the return will be undefined. 

              
var x = 5;

function logVariableValue(value) {
  console.log(value);
}

logVariableValue(x);

Any object can be used as either an argument or a return: Boolean, String, Number, Array, Object, Function... 

              
var x = 5;

function logVariableValue(value) {
  console.log(value);
  return(true);
}

logVariableValue(x);  // outputs 5
console.log(logVariableValue(5));  // outputs true

First class functions

Functions are First Class

  • functions may be passed into other functions as parameters
  • functions may have a return value that is a function
  • functions may be assigned to variables
  • functions may be stored in data structures

Functions are First Class

The fact that JavaScript treats functions as first-class enables the creation of what are termed "higher order" functions—functions that accept another function as input, output a function as return, or both.

This is a core tenet of functional programming.

Closures are Awesome!

A function in JavaScript that is itself a property of another function, and therefore has access to that function's properties, is known as a closure. Closures are great things when you need to create something approximating private method functions or variables.

This is a core tenet of object oriented JavaScript.

Closures are Awesome!

An additional use of closures is for namespacing your code.

This is super-helpful if you are working with a team on a project, and you don't want your code to pollute the global scope.

Closures are Awesome!

CodePen Example: http://codepen.io/dhezlrazorfish/pen/NqzVQe

Closures are Awesome! Suck!

As powerful as closures can be, they can also be incredibly frustrating if you're not careful.

The most important thing to note is that a closure doesn't store the actual value of its parent's properties, but rather a pointer to those properties. That is to say, if the value of the parent's property changes, then the value pulled into the closure likewise changes.

Closures are Awesome! Suck!

One such trap that developers frequently fall into is when attempting to declare functions inside of loops, which accidentally creates closures.

Closures are Awesome! Suck!

The below code would fail miserably when executed, due entirely to this error. In this case, creation of an anonymous function within the loop inadvertently creates a closure. Since the variable i is declared in the parent function, then all closures created are actually making use of a pointer to it in the parent.

              
for(var i = 1; i <= 10; i ++) {
  setTimeout(function() {
    console.log(i);
  }, 0);
}

Closures are Awesome! Suck!

So how do we fix it? One way would be to not create an anonymous function within our loop, instead invoking a function outside of the loop:

              
function report(value) {
  console.log(value);
}

for(var i = 1; i <= 10; i ++) {
  setTimeout( report(i), 0 );
}

Hoisting

JS will, at runtime, pull any declarations to the top of the containing obejct, while leaving any assignments in place. This is called hoisting.

Hoisting

Right now, the idea of this is probably making your skin crawl. With good reason. Hoisting can be a pain in the $%* if it catches you unawares, and can lead to some prodigiously sloppy code as developers move things around to try and deal with their assignments not getting hoisted.

Hoisting

              
(function(){
  
  doesHoist();
  doesNotHoist();
  
  function doesHoist() {
    console.log("hoists!");
  }
  
  doesNotHoist = function() {
    console.log("does not hoist!");
  }
  
})();

Hoisting

This works with other declarations/assignments just as it does with functions.

              
(function(){
  
  doesHoist = "does";
  
  console.log(doesHoist);
  console.log(doesNotHoist);
  
  var doesNotHoist = "does not";
  var doesHoist;
  
})();
