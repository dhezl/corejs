JavaScript in 2000

Fifteen years ago, you were liable to regularly encounter JavaScript in only a few scenarios:

  • Rollover state changes
  • DOM element click binding
  • Cookies
  • DHTML (remember that?!)
  • ...probably not a hell of a lot else

JavaScript in 2015

  • AMD and CommonJS help modularize code
  • Preprocessors allow people control over how they write code
  • Node.js has put JavaScript on the server
  • Node libraries can be used to drive microprocessors such as Arduino
  • Server-side practices, such as unit testing and continuous integration, have started flowing back into the presentation-layer JS community

JavaScript has become the lingua franca of the Internet

Good news, everybody!

Functions

Functions are a special type of object that is designed to perform a task. A function is executed when it is invoked by calling it using the () operator.

Functions can have parameters passed into them for consideration during execution. In defining a function, these variables are called parameters. The values themselves are known as arguments.

Functions return a value to their caller; this can be manual or automatic. If there is no return value specified, then the return will be undefined.

Any object can be used as either an argument or a return: Boolean, String, Number, Array, Object, Function... 

								
function foo( parameter1, parameter2, parameter3, ...) {
	return(true);
}

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
